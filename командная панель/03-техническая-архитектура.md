## Паттерны агрегации данных

### ETL через n8n (рекомендуется для малого бизнеса)

Для 5-10 источников данных ETL через n8n — оптимальный выбор. Данные трансформируются в Code-нодах и загружаются уже чистыми.

```
Паттерн 1: Schedule → Extract → Transform → Load
  [Cron 15min] → [HTTP Request: Bitrix24] → [Code: parse] → [Postgres/Sheets]

Паттерн 2: Webhook → Process → Notify
  [Webhook from CRM] → [Code: analyze] → [IF changed] → [Telegram alert]

Паттерн 3: Fan-out Aggregation
  [Cron daily] → [Split: sources] → [HTTP: each API] → [Merge] → [Sheets + DB]
```

### Real-time vs Batch

| Паттерн | Задержка | Применение | Реализация |
|---------|----------|------------|------------|
| **Real-time (push)** | <1 сек | Уведомления, алерты | n8n webhook → Redis → WebSocket |
| **Near real-time** | 1-15 мин | Метрики CRM, финансы | n8n schedule (каждые 15 мин) |
| **Batch (daily)** | 24 часа | Отчёты, аналитика | n8n cron (03:00 UTC) |

**90% данных Estate Invest подходят для batch (ежедневно) или near real-time (каждые 15-60 мин).**

## Слой хранения

### Google Sheets как "база данных"

| Параметр | Значение |
|----------|----------|
| Лимит ячеек | 10 млн на файл |
| Практический лимит | ~10 000 строк (комфортно), до 100 000 (медленно) |
| API лимит | 300 запросов/мин |
| gviz CSV | Быстрое чтение без API-ключа (public sheets) |

**gviz CSV хак**:
```
https://docs.google.com/spreadsheets/d/{SHEET_ID}/gviz/tq?tqx=out:csv&sheet={SHEET_NAME}
```
Поддерживает SQL-подобные запросы: `&tq=SELECT A,B,C WHERE D > 1000`

### SQLite vs PostgreSQL vs Supabase

| Параметр | SQLite | PostgreSQL | Supabase |
|----------|--------|------------|----------|
| Стоимость | Бесплатно | Бесплатно (self-host) | $0 free / $25/мес |
| Concurrency | 1 writer | Сотни клиентов | PostgreSQL + Realtime |
| Real-time | Нет | LISTEN/NOTIFY | WebSocket из коробки |
| n8n интеграция | Нет | Есть PostgreSQL нода | HTTP Request |

**Рекомендация**:
- Этап 1: Google Sheets (уже работает, данных <10К строк)
- Этап 2: PostgreSQL на VPS (единый бэкенд для Grafana/Metabase)

## Интеграции

### Битрикс24 REST API

**Endpoint**: `https://estateinvest.bitrix24.ru/rest/1/qlmnwa8sza8pvx14/{method}`

**Rate limits**: 2 запроса/сек, буфер 50 запросов

**Какие данные тянуть**:
```
crm.deal.list    → сделки (воронки, суммы, стадии)
crm.contact.list → контакты (кол-во, сегментация)
crm.lead.list    → лиды (конверсия, источники)
crm.activity.list → активность (звонки, письма)
```

**Паттерн n8n**:
```
[Schedule: каждые 60 мин]
  → [HTTP Request: POST crm.deal.list]
     body: { "filter": {">=DATE_MODIFY": "{{$now.minus(1,'hour').toISO()}}"} }
  → [Code: агрегация по воронкам и стадиям]
  → [Postgres/Sheets: UPSERT]
  → [IF: аномалия] → [Telegram: алерт]
```

**Пагинация**: start=0, start=50... (не превышать 2 req/сек)

### Google Sheets API

| Подход | Скорость | Аутентификация | Использование |
|--------|----------|----------------|---------------|
| **gviz CSV** | Быстрее | Нет (public) | Только чтение |
| **Sheets API v4** | Медленнее | OAuth/Service Account | Чтение + запись |

### n8n → Dashboard (обновление данных)

**Паттерн 1: Push в БД (простой, покрывает 95% задач)**
```
n8n workflow → [Postgres: INSERT метрики]
Grafana → читает из PostgreSQL (auto-refresh 30 сек)
```

**Паттерн 2: Push в Sheets → BI**
```
n8n workflow → [Google Sheets: append]
DataLens/Looker → читает из Sheets (кэш ~5 мин)
```

## Docker Compose стек (для VPS)

```yaml
# docker-compose.yml — командный центр на Hetzner CX22 ($5/мес)
version: '3.8'

services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: dashboard
      POSTGRES_USER: dashboard
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - pgdata:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  redis:
    image: redis:7-alpine
    command: redis-server --requirepass ${REDIS_PASSWORD}
    ports:
      - "6379:6379"

  grafana:
    image: grafana/grafana-oss:latest
    environment:
      GF_DATABASE_TYPE: postgres
      GF_DATABASE_HOST: postgres:5432
      GF_SECURITY_ADMIN_PASSWORD: ${GRAFANA_PASSWORD}
    ports:
      - "3000:3000"
    depends_on:
      - postgres

  caddy:
    image: caddy:2-alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data

volumes:
  pgdata:
  caddy_data:
```

Весь стек: ~$5/мес на Hetzner CX22 (2 vCPU, 4 GB RAM).
