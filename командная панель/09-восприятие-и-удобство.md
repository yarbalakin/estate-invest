# Восприятие и удобство — Как 3 партнёра читают дашборд

> 25.02.2026. Исследование: когнитивная нагрузка, eye-tracking, actionability.

---

## ГЛАВНЫЙ ПРИНЦИП: Дашборд отвечает на вопросы, а не показывает данные

Avinash Kaushik (Google): большинство дашбордов — это "customized data pukes" (красиво упакованные числа без контекста и рекомендаций).

**Дашборд должен отвечать на 3 вопроса:**
1. **Где мы?** (текущий статус)
2. **Где должны быть?** (план/цель)
3. **Что требует внимания?** (действия)

Вместо "Выручка: 15.3 млн" → "Выручка: 15.3 млн (+12% vs план). Рост за счёт 3 новых объектов."

---

## 1. ТЕСТ 5 СЕКУНД

Партнёр открыл дашборд. Через 5 секунд закрыл. Что запомнил?

Должен запомнить:
- **Бизнес ОК или нет** — общий статус (зелёный/жёлтый/красный)
- **Ключевая цифра** — North Star metric, крупно, вверху слева
- **Тренд** — вверх/вниз/стабильно (стрелка или спарклайн)

Если через 5 секунд не может ответить "бизнес в порядке или нет" — дашборд провалился.

**Как тестировать:** показать человеку, который не видел дашборд. Через 5 секунд спрятать. Спросить главный вывод.

## 2. КУДА СМОТРИТ ГЛАЗ (eye-tracking)

### F-паттерн — основной для дашбордов

Исследование Tableau (eye-tracking с 50+ участниками):

```
████████████████████████████   ← 1. Горизонтально по верху
████████████                   ← 2. Вниз по левому краю
████████████████████           ← 3. Ещё горизонтальное сканирование
██████                         ← 4. Внимание падает к низу
████
```

**Ключевые выводы:**
- Верхний левый угол = максимум внимания. **Самая важная цифра — туда.**
- Числа привлекают больше внимания, чем графики
- Сгруппированные элементы замечают чаще, чем разбросанные
- Нижний правый угол — "мёртвая зона"

**Практика:**
- North Star metric → верхний левый, шрифт в 2-3 раза крупнее остальных
- Проблемы/алерты → верхняя половина экрана (не прятать внизу)
- Детали и drill-down → нижняя половина

## 3. КОГНИТИВНАЯ НАГРУЗКА — Не 7, а 4

### Miller (1956) vs Cowan (2001)

| Автор | Год | Лимит |
|-------|-----|-------|
| Miller | 1956 | 7 ± 2 элементов |
| **Cowan** | **2001** | **3-5 чанков (в среднем 4)** |
| Cowan | 2010 | Подтвердил: "Magical Mystery Four" |

Miller's Law пересмотрен. Реальный лимит рабочей памяти — **4 чанка**, не 7.

**Чанк** — группа связанных данных. "Финансы компании" — это чанк. "Выручка + расходы + прибыль + ROI" по отдельности — 4 чанка.

### Что это значит для дашборда

- **4 крупных секции** на главном экране (не 9)
- В каждой секции — **2-3 метрики** максимум
- Итого: 4 × 3 = **12 метрик — верхний предел**
- Оптимально: 4 × 2 = **8 метрик**
- Партнёры смотрят "между делами" (стресс, мультитаскинг) → ёмкость ещё ниже

### 4 чанка для Estate Invest

| Чанк | Что внутри | Метрики |
|------|-----------|---------|
| **Финансы** | Деньги компании | Прибыль инвесторам, Капитал, ROI |
| **Объекты** | Недвижимость | Активные, Завершённые, Проблемные |
| **Инвесторы** | Люди | Активных, Новых, Сделки в воронке |
| **Внимание** | Что делать | Алерты с действиями |

## 4. NORTH STAR METRIC — Одно число вверху

### Концепция

Одна метрика, которая лучше всего отражает здоровье бизнеса. Отображается крупно, вверху слева.

**Как делают лидеры:**
- Stripe → Total Payment Volume
- Shopify → GMV мерчанта
- Airbnb → Забронированных ночей
- HubSpot → Weekly Active Teams

### Кандидаты для Estate Invest

| Метрика | За | Против |
|---------|-----|--------|
| **Прибыль инвесторам** (37 млн) | Показывает "здоровье", понятна всем | Медленно меняется |
| **ROI портфеля** (средневзвешенный) | Одно число, сравнимо с рынком | Сложная формула |
| **Активные объекты** (N из 118) | Простое, наглядное | Не отражает финансы |

**Рекомендация:** Прибыль инвесторам нарастающим итогом — это "пульс" бизнеса.

### Как отображать North Star

```
ПРИБЫЛЬ ИНВЕСТОРАМ      37.2 млн ₽    ▲ +8% к плану
████████████████████░░░   72% от годовой цели
[спарклайн ▁▂▃▃▄▅▅▆▇█]   12 месяцев
```

Tufte: "Число без тренд-линии бессмысленно. Циферблаты — пустая трата пикселей."

## 5. ПРИНЦИП СВЕТОФОРА (RAG)

### Red / Amber / Green

- **Зелёный** — всё по плану, цели достигаются
- **Жёлтый** — потенциальные проблемы, незначительные отклонения
- **Красный** — критические проблемы, нужно действовать

### Правила

1. **Не злоупотреблять** — RAG на 5-7 ключевых показателях, не на 50
2. **Чёткие пороги** — заполняемость >85% = зелёный, 70-85% = жёлтый, <70% = красный
3. **Дублировать символами** — для дальтоников (~8% мужчин): ✓ / ⚠ / ✗
4. **Сопровождать текстом** — почему красный и что делать

### Пороги для Estate Invest

| Метрика | Зелёный | Жёлтый | Красный |
|---------|---------|--------|---------|
| ROI объекта | > 15% | 8-15% | < 8% |
| Срок реализации | В плане | Задержка < 30 дней | Задержка > 30 дней |
| Сбор средств | > 90% | 70-90% | < 70% |
| Обработка анкеты | < 24ч | 24-48ч | > 48ч |

## 6. КОНТЕКСТ СРАВНЕНИЯ — "По сравнению с чем?"

Tufte: дашборды бесполезны без истории. "Compared to what?"

### 4 типа контекста

| Тип | Пример | Когда |
|-----|--------|-------|
| **vs Цель** | 15 млн из 20 млн (75%) | Есть утверждённый план |
| **vs Прошлый период** | +12% к прошлому месяцу | Тренды |
| **vs Год назад** | +8% YoY | Устраняет сезонность |
| **vs Бенчмарк** | ROI 18% vs рынок 12% | Позиционирование |

### Шаблон отображения каждой метрики

```
Капитал    196 млн ₽    ▲ +12% за месяц    [▁▃▅▆▇ спарклайн]
           ████████████████░░░░  82% от цели (240 млн)
```

**Правило:** каждая метрика на дашборде ОБЯЗАНА иметь хотя бы один тип сравнения. Голое число — бесполезное число.

## 7. ACTIONABILITY — Красное = действие

### Проблема

Красный индикатор без рекомендации — просто тревога. Партнёр видит "плохо" но не знает что делать.

### Формула Kaushik: Insight → Recommendation → Impact

| Компонент | Пример |
|-----------|--------|
| **Insight** | Объект #47 — задержка ремонта 12 дней |
| **Рекомендация** | Связаться с подрядчиком Ивановым |
| **Impact** | Потеря 400К/мес при текущей задержке |

### Практика для дашборда

Не просто:
```
❌  Объект #47 — КРАСНЫЙ
```

А так:
```
✅  ! Объект #47 Арбат — задержка ремонта 12 дней
    → Связаться с подрядчиком (отв: Иванов)
    → При текущем темпе: -400К/мес
```

**Блок "Требует внимания"** — на главном экране, верхняя половина. Максимум 3-5 пунктов. Каждый с действием и ответственным.

## 8. МОБИЛЬНЫЙ ВИД — 5 секунд между встречами

### Принцип

Мобильный вид — это НЕ уменьшенный десктоп. Это **отдельный** интерфейс.

### Правила

- Максимум **3-4 карточки** на экране
- **Одна колонка**, вертикальный скролл
- Шрифт KPI: **24-32px** минимум
- Touch targets: **44×44px** минимум (Apple HIG)
- Много воздуха между элементами

### Минимальный мобильный вид для партнёров

```
┌─────────────────────────────┐
│  ESTATE INVEST    [● ОК]    │
├─────────────────────────────┤
│                             │
│  Прибыль инвесторам         │
│  37.2 млн ₽    ▲ +8%       │
│  ███████████████░░░  72%    │
│                             │
├─────────────────────────────┤
│  Объекты         Инвесторы  │
│  94 активных     912        │
│  3 проблемных    +14/мес    │
├─────────────────────────────┤
│  Требует внимания (3)       │
│  ! Объект #47 — 12 дн.     │
│  ! Петров — вывод 2.1М     │
│  ! #83 ROI 4.2% < 8%       │
└─────────────────────────────┘
```

5 показателей + 3 алерта. Детали — по тапу. Всё.

## 9. СПЕЦИФИКА ESTATE INVEST

### Метрики по уровням

**Tier 1 — Главный экран (8 метрик):**

| # | Метрика | Зачем |
|---|---------|-------|
| 1 | Прибыль инвесторам (North Star) | Здоровье бизнеса |
| 2 | Капитал под управлением | Масштаб |
| 3 | ROI портфеля (средневзвешенный) | Эффективность |
| 4 | Активных объектов | Операционная нагрузка |
| 5 | Проблемных объектов | Что горит |
| 6 | Активных инвесторов | База |
| 7 | Сделки в воронке | Pipeline |
| 8 | Общий статус (RAG) | Моментальная оценка |

**Tier 2 — По клику (drill-down):**
- Cash Flow по месяцам
- Канбан объектов по стадиям
- Динамика регистраций
- Конверсия воронки
- Мультивалютный баланс

**Tier 3 — Детали (отдельные страницы):**
- Полная таблица объектов с фильтрами
- Карточки инвесторов
- P&L по объекту
- Календарь торгов

### Портфельный вид — не 118 строк

Для 118 объектов — агрегация, не список:

```
ОБЪЕКТЫ ПО СТАДИЯМ

Подготовка    ██░░░░░░░░  12
Сбор          ████░░░░░░  28
Реализация    ██████████  58    ← основная масса
Продажа       ███░░░░░░░  17
Завершено     ██░░░░░░░░  3

⚠ 3 проблемных (клик для деталей)
```

```
║  4.2 млн pipeline     ║    → Подготовить документы          ║
║                       ║                                    ║
║                       ║  ! #83 ROI 4.2% (план 8%)          ║
║                       ║    → Пересмотреть цену              ║
╚═══════════════════════╩════════════════════════════════════╝
```

### Почему именно так

| Решение | Основание |
|---------|-----------|
| 4 блока | Cowan: лимит рабочей памяти = 4 чанка |
| North Star вверху слева | Eye-tracking: максимум внимания |
| RAG-статус в заголовке | 1-секундный ответ на "бизнес ОК?" |
| Каждая метрика + дельта | Tufte: число без контекста бесполезно |
| Алерты с действиями | Kaushik: insight → recommendation → impact |
| Прибыль = North Star | Отражает ценность для клиентов (инвесторов) |
| 8 метрик на экране | 4 блока × 2 метрики = оптимум по Cowan |
| Drill-down по клику | Progressive Disclosure (NN/Group) |
