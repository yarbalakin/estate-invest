# UX-паттерны для CEO-дашборда — Глубокое исследование

> Собрано: 25.02.2026. Навигация, когнитивная нагрузка, мобильные, антипаттерны.

---

## 1. НАВИГАЦИЯ

### Sidebar — лучший выбор для 10+ секций

- Пользователи смотрят на левую половину экрана **80% времени** (NN/Group)
- Вертикальные списки сканируются эффективнее горизонтальных
- Оптимальная ширина: **240-300px** развёрнутый, **48-64px** свёрнутый

### Управление сложностью

**Сворачиваемые группы**: категории раскрываются по клику (как Google Analytics)
**Избранное**: закреплённые секции наверху (как DataDog)
**Поиск**: строка фильтра при 10+ секциях
**Breadcrumbs**: вторичная навигация, показывает иерархию (не историю)

### Когда что использовать

| Паттерн | Когда |
|---------|-------|
| Sidebar | 10+ секций, основная навигация |
| Top-nav | < 5 пунктов |
| Tabs | 3-7 однородных вкладок, сравнение |

## 2. ИЕРАРХИЯ ДАННЫХ

### Закон Миллера (7 +/- 2)

- **5-7 KPI** на главном экране — абсолютный максимум
- Верхний левый квадрант = максимум внимания
- "Data isn't like your kids. You don't have to pretend to love them equally" — Amanda Cox, NY Times

### Progressive Disclosure (3 уровня)

| Уровень | Что видит CEO | Время |
|---------|--------------|-------|
| **Glanceable** | 5-7 KPI с числами, стрелками, цветом | 3 сек |
| **Explorable** | Клик → тренд (график), разбивка | 30 сек |
| **Deep dive** | Полная таблица, фильтры, экспорт | 2+ мин |

### 4 когнитивных принципа (UX Magazine)

1. **Читаемость**: правильный контраст, не все данные равны
2. **Минимизация нагрузки**: вся инфо на одном экране, recognition > recall
3. **Графики > таблицы** для восприятия трендов
4. **Законы гештальта**: простые паттерны → быстрая обработка

## 3. УПРАВЛЕНИЕ ПЕРИОДАМИ

### 3 паттерна

**Пресеты (лучший для CEO)**: Сегодня / Неделя / Месяц / Квартал / Год / Всё время — один клик
**Календарный picker**: дополнение к пресетам, для точных дат
**Относительные**: "Последние 30 дней", YTD, QTD — автообновляются

### Сравнение периодов

Всегда показывать: абсолютное изменение + процент + стрелка + цвет
- vs предыдущий период
- vs аналогичный период прошлого года

### Обновление данных

| Метод | Когда |
|-------|-------|
| Polling (setInterval) | Раз в 30-60 сек, простейший |
| SSE | Односторонний push, идеально для дашбордов |
| WebSocket | Избыточно для dashboard-only |

Рекомендация: **polling каждые 60 секунд** — достаточно для CEO.

## 4. ЗАГРУЗКА И ПУСТЫЕ СОСТОЯНИЯ

### Skeleton Screens

| Время ожидания | Решение |
|----------------|---------|
| < 1 сек | Ничего |
| 2-10 сек | Skeleton screen или spinner |
| > 10 сек | Progress bar |

**Shimmer-эффект**: градиент с анимацией `background-position`. Для полной загрузки страницы. Спиннеры — для отдельных виджетов.

### Пустые состояния — 3 принципа

1. **Статус**: "Нет записей для выбранного периода" (не пустой экран)
2. **Обучение**: "Добавьте избранные метрики, чтобы видеть их здесь"
3. **Действие**: CTA-кнопка "Создать" / "Добавить" / "Настроить"

Формулировки: **позитивные** ("Начните с..."), не негативные ("У вас нет...").

## 5. ACCESSIBILITY В DARK MODE

### Контраст (WCAG AA)

| Элемент | Минимум |
|---------|---------|
| Обычный текст | 4.5:1 |
| Крупный текст (18px+ / 14px bold) | 3:1 |
| UI-компоненты | 3:1 |

### Ключевые правила

- **Никогда #000000** — вызывает halation. Используй #121212 / #0f172a
- **Текст не #FFFFFF** — слишком резкий. Используй #E0E0E0 / #f1f5f9
- **Font-weight minimum 400** — thin/light "исчезает" на тёмном
- **Увеличить letter-spacing** и line-height vs светлая тема
- **Десатурировать** яркие цвета на 10-20% для тёмной темы
- **Фокус-индикатор** чётко виден на тёмном фоне (outline, не цвет)

## 6. МОБИЛЬНЫЕ ПАТТЕРНЫ

### Принципы для CEO на телефоне

- CEO на мобильном = "между встречами" — нужен ответ за **5 секунд**
- На мобильном — **только ключевые KPI**, не весь десктоп
- 1 колонка, карточки стекаются вертикально

### Breakpoints

| Экран | Колонки | Что видно |
|-------|---------|----------|
| Desktop 1200px+ | 3-4 | Полная сложность |
| Tablet 768-1199px | 2 | Часть скрыта |
| Mobile < 768px | 1 | Только KPI + ключевые графики |

### Touch

- Минимальная кнопка: **44x44px** (Apple) / **48x48px** (Material)
- Свайп: горизонтальный между секциями
- Sidebar: скрытый, по гамбургеру

## 7. СТОРИТЕЛЛИНГ С ДАННЫМИ

### Нарратив в дашборде

Дашборд — "выражение приоритетов". Выбор метрик уже рассказывает историю.

### Три акта

1. **Setup**: что это за данные, почему важно
2. **Confrontation**: аномалии, тренды, отклонения от целей
3. **Resolution**: какие действия предпринять

### Практические приёмы

- **Hero-метрики** = показатели успеха, **Villain-метрики** = проблемы
- **Аннотации на графиках** — отмечать события: запуск, сбой, изменение стратегии
- **Контекст рядом** — цели, бенчмарки, временные рамки — не заставлять искать
- **Branching narratives** — дашборд = "choose your own adventure"

## 8. АНТИПАТТЕРНЫ — ЧТО НЕ ДЕЛАТЬ

### Топ-8 ошибок

| # | Ошибка | Решение |
|---|--------|---------|
| 1 | **Информационная перегрузка** — стена данных | Max 5-7 KPI, progressive disclosure |
| 2 | **Данные без контекста** — число без "много или мало?" | Всегда: vs прошлый период, vs цель, тренд |
| 3 | **Неправильный тип визуализации** — pie для 15 категорий | Тренд=линия, сравнение=бары, доля=donut до 5 |
| 4 | **Идеальная симметрия** — все блоки одинаковые | Размер блока = его важность |
| 5 | **Один дашборд для всех** | Ролевая кастомизация |
| 6 | **Не интегрирован в процесс** | "Домашняя страница" рабочего дня |
| 7 | **Цветовой хаос** — радуга без системы | 2-3 основных + семантические |
| 8 | **Нет действия** — видно проблему, но нет пути | Кнопка/ссылка рядом с алертом |
