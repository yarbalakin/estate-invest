## 1. Концепция "War Room" в бизнесе

Бизнес-термин заимствован из военных штабов и NASA Mission Control. Суть: **централизация информации, людей и решений в одной точке** кратно повышает скорость и качество решений.

### Три принципа

1. **Single Source of Truth** — единый источник правды. Дашборд устраняет конфликтующие отчёты
2. **Радикальная прозрачность** — не прячет проблемы. Обсуждения на объективной реальности
3. **Скорость решений** — минимизация времени от обнаружения проблемы до действия

### "Information Radiator" (Alistair Cockburn, 2001)

Информация должна **излучаться** — видна всем без специальных усилий. Для Estate Invest (10 человек) — это HTML-дашборд на постоянно открытой вкладке + Telegram-алерты.

## 2. Data Storytelling: Пирамида Минто для дашбордов

Barbara Minto (McKinsey, 1970-е):

**Пирамида на экране:**
- **Вершина** — главный вывод (самая крупная цифра)
- **Средний уровень** — поддерживающие аргументы (3-5 KPI)
- **Основание** — детальные данные (drill-down)

**SCQA-фреймворк для Estate Invest:**
- **S**ituation: "118 объектов, 900+ инвесторов"
- **C**omplication: "Средний срок продажи вырос на 15 дней"
- **Q**uestion: "Какие объекты тормозят конвейер?"
- **A**nswer: таблица с конкретными объектами и действиями

### Аудитория определяет историю
- **Руководитель** — "Мы заработали / потеряли X"
- **Аналитик** — "Почему тренд изменился"
- **Операционист** — "Какой объект обработать следующим"

## 3. Анти-паттерны дашбордов (12 ошибок)

1. **Свалка данных** — дашборд не отчёт. Только ценные выводы
2. **Цифры без контекста** — "47 сделок" бессмысленно без сравнения с планом/прошлым
3. **Злоупотребление цветами** — макс 2-3 цвета, 8% мужчин — дальтоники
4. **RAG-светофор** — люди ненавидят "красный" рядом с именем и манипулируют. Альтернатива: XmR-графики
5. **3D-графики** — невозможно точно прочитать данные
6. **Один дашборд для всех** — разные роли = разные потребности
7. **Нет пустого пространства** — белое пространство помогает "переварить"
8. **Неправильный тип графика** — pie для 12 категорий, bar для timeline
9. **Нет фокуса** — дашборд без цели = декорация
10. **Устаревшие данные** — вредит больше, чем помогает
11. **Vanity metrics** — "900 инвесторов" (vanity) vs "Конверсия 37%" (actionable)
12. **Игнорирование верхнего левого угла** — глаз начинает оттуда

## 4. "One Metric That Matters" (OMTM)

### Фреймворк (Alistair Croll, "Lean Analytics")

В каждый момент времени есть ОДНА метрика важнее всех. OMTM меняется каждые 2-4 месяца.

### North Star Metric (NSM)

Долгосрочная "путеводная звезда", отражающая ценность для клиента. OMTM = тактический фокус, NSM = стратегический ориентир.

### AARRR "Пиратские метрики" для Estate Invest

| Стадия | Метрика | Текущее значение |
|--------|---------|-----------------|
| **Acquisition** | Новые лиды | 1,343 в CRM |
| **Activation** | Первый звонок / встреча | — |
| **Retention** | Повторные инвестиции | Ключевое! |
| **Revenue** | Средний чек инвестиции | 40K-1.6M |
| **Referral** | Инвесторы по рекомендации | 10% программа |

**Кандидаты на NSM**: "Сумма средств инвесторов под управлением" или "Количество активных инвестиций"

**OMTM прямо сейчас**: если проблема продажи — "Конверсия воронки Регистрация: 37%" с целью 45%

## 5. Философия алертинга

### Три типа алертов

| Тип | Когда | Пример для Estate Invest |
|-----|-------|--------------------------|
| **Threshold** | Метрика за границами | "Активные объекты < 100" |
| **Anomaly** | Отклонение от паттерна | "Трафик сайта -60% без причины" |
| **Trend** | Изменение направления | "Срок продажи растёт 3 недели подряд" |

### Когда алертить vs показывать на дашборде

**Авиационный принцип**: красный алерт — только если пилот ДОЛЖЕН действовать СЕЙЧАС. Всё остальное — на приборную панель.

**Для Estate Invest**: макс 3-5 алертов в день в Telegram. Каждое повторное уведомление снижает внимание на 30%.

### Борьба с Alert Fatigue
- **Специфичность**: "Объект Калининград-15: цена ниже рынка на 20%, нет просмотров 7 дней" (хорошо) vs "Что-то не так" (плохо)
- **Парные метрики** (Andy Grove): каждая метрика + контрметрика
- **Аудит**: раз в месяц пересматривай все алерты

## 6. Иерархия дашбордов

### Три уровня

| Уровень | Кто | Горизонт | Обновление |
|---------|-----|----------|------------|
| **Стратегический** | CEO/собственник | Месяцы, годы | Ежедневно |
| **Тактический** | Руководители | Недели, месяцы | Ежедневно |
| **Операционный** | Исполнители | Часы, дни | Real-time |

### Для команды из 10 человек

НЕ нужны 3 отдельных дашборда. Нужен ОДИН с тремя "этажами":
- **Верх**: 3-5 стратегических метрик (NSM + OMTM + финансы)
- **Середина**: операционные метрики по направлениям
- **Низ / вторая страница**: детализация с drill-down

## 7. Ключевые фреймворки и мыслители

### Фреймворки

| Фреймворк | Автор | Суть | Для Estate Invest |
|-----------|-------|------|-------------------|
| **Balanced Scorecard** | Kaplan & Norton, 1992 | 4 перспективы: Финансы, Клиенты, Процессы, Рост | 8-12 метрик на одной странице |
| **OKR** | Grove → Doerr | Цель + 3-5 Ключевых результатов | OKR-дашборд с прогресс-барами |
| **Lean Analytics / OMTM** | Croll, 2013 | Одна метрика на период | Максимальный эффект для малой команды |
| **Пирамида Минто** | Minto, 1970-е | Начни с ответа | Структура каждого экрана |

### Мыслители

**Edward Tufte** (отец визуализации):
- **Data-Ink Ratio**: максимизируй долю чернил, показывающих данные
- **Sparklines**: микрографики в тексте/таблице
- "Дашборд без истории бесполезен"

**Stephen Few** ("Information Dashboard Design"):
- "Отображай данные максимально ясно и просто"

**Andy Grove** ("High Output Management"):
- Измеряй output, не activity
- **Парные индикаторы**: скорость + качество
- **Leading vs Lagging**: не ждать плохих lagging, действовать на leading

**Jeff Bezos / Amazon**:
- **Input > Output metrics**: "Зацикливайся на input, это единственное, на что ты влияешь каждый день"

### Предупреждения

**Goodhart's Law**: "Когда метрика становится целью, она перестаёт быть хорошей метрикой"

**Противоядие**: парные метрики, не привязывай KPI к наказаниям, регулярно пересматривай

## 10 принципов для командного центра Estate Invest

1. **Определи NSM** — "Сумма активных инвестиций" или "Объекты в обороте"
2. **OMTM на квартал** — одна метрика, которую вся команда знает
3. **Пирамида Минто на экране** — верх = NSM, середина = KPI, низ = drill-down
4. **Sparklines везде** — число + микрограф тренда (Tufte)
5. **Input > Output** (Bezos) — "показов инвесторам/нед" > "выручка"
6. **Парные метрики** (Grove) — "Скорость продажи" + "Маржа"
7. **Exception-based алерты** — в Telegram только действия
8. **Макс 3-5 алертов/день** — иначе alert fatigue
9. **Balanced Scorecard lite** — Финансы, Инвесторы, Процессы, Рост
10. **Анти-Goodhart** — метрики для понимания, не для наказания
