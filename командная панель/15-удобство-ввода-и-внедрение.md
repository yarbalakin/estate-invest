# Удобство ввода данных и лёгкость внедрения

> Принцип: командный центр ровно настолько хорош, насколько легко людям заполнять данные. Если заполнять неудобно — данные будут грязные, неполные, и дашборд бесполезен.

---

## Текущая ситуация: где команда вводит данные СЕЙЧАС

| Кто | Что вводит | Куда | Как часто | Удобство (1-5) |
|---|---|---|---|---|
| **Александра** (статистика) | Транзакции, расчёты | 117 отдельных Google Sheets | Ежедневно | 2 — ручной копипаст между таблицами |
| **Ирина, Наталия, Снежана** (поддержка) | Обращения инвесторов, статусы | Битрикс24 CRM + Salebot | Ежедневно | 3 — Битрикс громоздкий, но привычный |
| **Кирилл, Антонина, Татьяна** (аналитика) | Объекты с торгов, скрининг | Google Sheets (таблица торгов) | Несколько раз в неделю | 3 — вручную, но понятный формат |
| **Андрей, Наталья** (реализация) | Показы, звонки, результаты | Нигде систематически (!) | По ситуации | 1 — данные теряются |
| **Иван** (объекты) | Статус ремонта, расходы | Google Sheets (касса объекта) | По событию | 2 — нужно найти нужный из 117 sheets |
| **Глеб** (контент) | Посты, рассылки | Контент-план (Sheets) | Еженедельно | 3 — стандартный процесс |

### Главные боли

1. **117 отдельных Google Sheets** — нужно найти нужный, открыть, вспомнить формат, вписать. С телефона — мучение
2. **Данные о показах НЕ ФИКСИРУЮТСЯ** — менеджер провёл 5 показов, нигде не записал → нет аналитики продаж
3. **Финансы НЕ в Битрикс24** — CRM знает про сделки, но суммы только в Sheets → два источника правды
4. **Нет мобильного ввода** — в полях (показ объекта, осмотр) негде быстро записать результат

---

## Принцип: данные должны вводиться ТАМ, ГДЕ ЧЕЛОВЕК УЖЕ НАХОДИТСЯ

| Где человек | Что удобно вводить | Инструмент |
|---|---|---|
| В Telegram (90% времени) | Быстрые заметки, суммы, фото, статусы | Telegram-бот |
| За компьютером | Таблицы, расчёты, массовый ввод | Google Sheets (привычные) |
| На показе объекта | Фото, результат, комментарий | Telegram-бот (телефон в руке) |
| В офисе | Работа с CRM, документами | Битрикс24 |

**Вывод**: для полевых сотрудников (показы, осмотры, расходы) — Telegram-бот. Для офисных (статистика, аналитика) — Google Sheets, который они уже знают.

## Решение 1: Telegram-бот "Помощник команды"

### Почему Telegram, а не приложение/форма

| Критерий | Telegram-бот | Google Forms | Мобильное приложение |
|---|---|---|---|
| Время до первого ввода | **2 сек** (уже в чате) | 10 сек (найти ссылку, открыть) | 5 сек (найти приложение) |
| Фото + текст одним потоком | Нативно | Отдельная загрузка | Зависит от приложения |
| Геолокация | Одна кнопка | Нет | Да |
| Установка | Не нужна | Не нужна | Нужна |
| Обучение | 0 (все в Telegram) | Минимальное | Среднее |
| Стоимость | $0 (n8n) | $0 | $5-60/мес (AppSheet/Glide) |

### Что делает бот (по ролям)

**Для менеджеров по реализации (Андрей, Наталья):**
```
/показ
→ Бот: "Какой объект?" → [ИП-031] [ИП-091] [ИП-092] ... (inline-кнопки)
→ Бот: "Результат?" → [Интерес] [Думает] [Отказ] [Сделка]
→ Бот: "Комментарий?" → свободный текст
→ Бот: "Фото?" → необязательно
→ Записывает в Google Sheets + уведомляет руководителя
```
Время: 20-30 секунд. Вместо: забыть → не записать → потерять данные.

**Для Ивана (работа с объектами):**
```
/расход
→ Бот: "Объект?" → [ИП-031] [ИП-091] ...
→ Бот: "Сумма?" → 15000
→ Бот: "Категория?" → [Ремонт] [Нотариус] [Госпошлина] [Транспорт] [Другое]
→ Бот: "Фото чека?" → необязательно
→ Записывает в кассу объекта (Google Sheets)
```
Вместо: найти нужный из 117 sheets → открыть → найти строку → вписать.

**Для аналитиков (Кирилл, Антонина):**
```
/объект [ссылка на лот]
→ Бот парсит страницу торгов автоматически
→ Бот: "Объект: ул. Ленина 5, 45м2, 2.1 млн. Добавить?" → [Да] [Нет]
→ Записывает в таблицу торгов + Битрикс24
```
Вместо: открыть страницу → скопировать данные → вставить в Sheets → заполнить вручную.

**Для всех:**
```
/статус
→ Показывает текущие KPI: объекты в работе, показы за неделю, деньги
/помощь
→ Список команд
```

### Техническая реализация

Уже есть все компоненты:
- **n8n cloud** — ядро (есть)
- **Telegram-бот** — опыт с Денежным ботом (есть)
- **Google Sheets** — хранение (есть)
- **Битрикс24 REST API** — синхронизация (есть)

По паттерну Денежного бота: Webhook → Switch → State Machine → Google Sheets + Telegram HTTP Request.

## Решение 2: "Сводный лист" вместо 117 отдельных

### Проблема 117 sheets

Сейчас каждый объект — отдельная таблица. Чтобы понять картину, нужно открыть десятки файлов. Это невозможно для дашборда.

### Стратегия "Shadow Sync" (не ломаем существующее)

```
117 отдельных касс (как есть, никто не замечает изменений)
         ↓ (Apps Script, ночью, автоматически)
    Мастер-таблица "Сводная"
    ├── Лист "Все транзакции" (нормализованные данные из 117 sheets)
    ├── Лист "Объекты" (статусы, суммы, стадии)
    └── Лист "Аномалии" (дубли, пропуски, ошибки формата)
         ↓
    DataLens / Looker Studio (дашборд)
```

**Что это даёт:**
- Сотрудники продолжают работать как привычно
- Никого не надо переучивать
- Данные автоматически собираются в одно место
- Дашборд строится поверх сводной таблицы

### Технические нюансы

- **IMPORTRANGE НЕ подходит** — лимит ~50 перекрёстных ссылок на таблицу, для 117 источников невозможно
- **Apps Script** — читает 117 sheets за 2-3 запуска по ~2 мин каждый (cursor + trigger chaining)
- **Google Sheets API** — 60 reads/min/user, 117 sheets за ~2 минуты
- **Запуск**: ежедневно ночью, не мешает пользователям
- **Стоимость**: $0

### Для новых объектов — шаблон

Все новые ИП создаются по шаблону с фиксированной структурой:
- Колонки в строгом порядке (Дата | Операция | Сумма | Категория | Комментарий)
- Data Validation — dropdown для категорий
- Protected headers — заголовки нельзя случайно изменить
- Скрипт создаёт новый sheet одной кнопкой

## Решение 3: Битрикс24 — использовать то, что есть

### Сейчас: CRM знает про сделки, но не знает суммы

Воронка "Регистрация" работает (425 WON, 37% конверсия). Но финансовые данные — только в Sheets.

### Что можно улучшить без дополнительных инструментов

**а) Мобильное приложение Битрикс24** — уже установлено у сотрудников?
- Можно обновлять стадию сделки прямо с телефона
- Быстрые звонки из CRM
- Push-уведомления о новых лидах

**б) Пользовательские поля в сделках:**
- Добавить поле "Сумма инвестиции" в воронку Регистрация
- Добавить поле "ИП номер" (привязка к объекту)
- Это позволит строить отчёты в самом Битрикс24 без внешних инструментов

**в) n8n → Битрикс24 sync:**
- Когда в Google Sheets появляется новая транзакция → n8n обновляет сделку в Битрикс24
- Результат: CRM содержит и воронку, и суммы → один источник

### Ограничение

Битрикс24 — не самый удобный инструмент для финансового учёта. Его сила — воронка продаж и коммуникация. Заставлять команду вести финансы в Битрикс = лишнее трение.

**Рекомендация**: оставить финансовый ввод в Sheets (привычно), но sync в Битрикс автоматически через n8n.

## Решение 4: Голосовой ввод

### Концепция

Менеджер на показе надиктовывает голосовое в Telegram:
> "Показ объекта ИП-91, Добрянка, клиент Иванов, интересуется, хочет посмотреть второй этаж, бюджет пять миллионов"

Бот через AI (OpenRouter, бесплатно):
1. Транскрибирует голос (Telegram API даёт ogg-файл)
2. AI извлекает: объект = ИП-91, клиент = Иванов, результат = интерес, бюджет = 5 млн
3. Подтверждает: "ИП-91, Иванов, интерес, 5 млн. Верно?" → [Да] [Нет]
4. Записывает

### Реализация

n8n шаблон существует: [Multi-Modal Expense Tracking with GPT-4, Gemini OCR, and Voice](https://n8n.io/workflows/8035-multi-modal-expense-tracking-with-gpt-4-gemini-ocr-and-voice-via-telegram/)

Принимает текст, фото чеков и голосовые. Всё парсится AI и пишется в Sheets.

### Когда полезно

- Менеджер за рулём → надиктовал отчёт
- На объекте без возможности печатать → голосовая заметка
- Для людей, которым лень нажимать кнопки

### Когда не нужно

- Для финансовых данных (нужна точность, голос = ошибки)
- Для массового ввода (офисная работа быстрее в Sheets)

## Чего НЕ делать

1. **Не заставлять команду менять привычки сразу** — Shadow Sync читает старые sheets, не требует переучивания
2. **Не строить дашборд до сбора данных** — сначала бот для показов (решение 1), потом сводная (решение 2), потом визуализация
3. **Не внедрять новые приложения** — AppSheet, Glide, Airtable требуют обучения. Telegram и Sheets команда уже знает
4. **Не усложнять бота** — начать с 2-3 команд (/показ, /расход, /статус), расширять по запросу
5. **Не делать real-time** — ежедневного обновления достаточно для недвижимости
6. **Не мигрировать на базу данных сразу** — Google Sheets как хранилище пока справляется при 76K строк

## Конкретный план внедрения (2 недели)

### Неделя 1: Данные о показах + Shadow Sync

**День 1-2**: Telegram-бот "Показы" (n8n)
- Создать бот через @BotFather
- n8n workflow: Webhook → State Machine → Google Sheets
- 3 команды: /показ, /расход, /статус
- Тестирование с 1-2 менеджерами

**День 3-4**: Shadow Sync (Apps Script)
- Скрипт чтения 117 касс (cursor + trigger chaining)
- Мастер-таблица с нормализованными данными
- Ежедневный автозапуск ночью

**День 5**: Проверка качества
- n8n workflow: утренний check на аномалии → Telegram
- Data Validation на 5-10 самых активных sheets

### Неделя 2: Визуализация + итерация

**День 6-7**: Подключить DataLens или Looker Studio
- К мастер-таблице (не к 117 отдельным!)
- 5-8 графиков: объекты по стадиям, cash flow, воронка, показы

**День 8-9**: Feedback от команды
- Показать дашборд партнёрам
- Спросить: "Что добавить? Что лишнее?"
- Пригласить менеджеров в бот

**День 10**: Корректировки
- Добавить/убрать метрики по фидбеку
- Настроить Telegram-алерты (максимум 3-5 в день)

## Ключевой тезис

**Не начинай с дашборда. Начинай с ввода данных.**

Самый красивый командный центр бесполезен, если менеджеры не записывают показы, а финансисту сложно найти нужную из 117 таблиц. Решив проблему ввода (Telegram-бот + Shadow Sync), получишь чистые данные, и дашборд построится за день.
