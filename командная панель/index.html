Layout, цвета, шрифты, KPI-карточки, иконки, микро-взаимодействия, референсы. 30+ источников.

Sidebar + Top Rail + Content Grid

Доминирующий паттерн 2025-2026 — сворачиваемый сайдбар слева (Vercel, Linear, Stripe) + горизонтальная полоса KPI сверху + основной контент в grid-сетке.

Count-up чисел (Vanilla JS)

function animateValue(element, start, end, duration) {
  const startTime = performance.now();
  function update(t) {
    const progress = Math.min((t - startTime) / duration, 1);
    const eased = 1 - Math.pow(2, -10 * progress);
    element.textContent = Math.floor(start + (end-start) * eased)
      .toLocaleString('ru-RU');
    if (progress < 1) requestAnimationFrame(update);
  }
  requestAnimationFrame(update);
}

07. Premium-эстетика

CSS-сниппеты и техники для "wow"-эффекта.

Noise/Grain (зернистость)

SVG feTurbulence делает интерфейс "осязаемым". baseFrequency 0.5-0.8, opacity 0.08-0.15.

Glow (свечение)

.card-glow:hover {
  box-shadow:
    0 0 0 1px rgba(255,255,255,0.1),
    0 8px 25px rgba(0,0,0,0.4),
    0 0 60px rgba(59,130,246,0.15);
}

Aurora-фон

Движущиеся блобы с filter: blur(80px). Правило: opacity 0.3-0.5 max, предельно тонкая.

Анимация чисел

function animateNumber(el, target, duration = 1500) {
  const start = performance.now();
  function update(t) {
    const p = Math.min((t - start) / duration, 1);
    const eased = 1 - Math.pow(1 - p, 4); // easeOutQuart
    el.textContent = Math.round(target * eased)
      .toLocaleString('ru-RU');
    if (p < 1) requestAnimationFrame(update);
  }
  requestAnimationFrame(update);
}

Premium-рецепт (чеклист)

Must-have (max эффекта, min кода)

  • Тёмный фон #0f172a + текст #f1f5f9
  • Noise-оверлей через SVG (opacity 0.1)
  • font-variant-numeric: tabular-nums
  • Glassmorphism: backdrop-filter: blur(12px)
  • Многослойные тени + glow на hover
  • Каскадный fadeInUp при загрузке
  • Анимация чисел при появлении
  • 1px бордеры opacity 0.05-0.1, radius 12-16px

Nice-to-have

  • Aurora-фон (тонкий, opacity 0.3)
  • 3D tilt карточек (threshold 5-8)
  • SVG progress rings
  • Skeleton shimmer при загрузке
Важно: Stripe, Linear и Vercel используют 2-3 эффекта, не все 14. "Less is more" — выбрать максимум 4-5 и сделать хорошо.

08. Критический разбор

Честный взгляд опытного разработчика на собранные материалы.

Главная проблема: исследование описывает три разных продукта одновременно — статичный HTML, SaaS-дашборд уровня Stripe и BI-платформу. Нужно выбрать одно.

10 проблем

1. Масштаб не соответствует реальности

50 виджетов для 2 человек. Must-have на 25 пунктов = wish list. Для MVP: 5-7 KPI + 3 графика + 1 таблица.

2. Single HTML file — ложь при таком скоупе

50 виджетов = 8-12K строк. Это уже приложение, а не HTML-файл. Либо несколько файлов, либо радикально сократить скоуп.

3. Данные — самое сложное, а не дизайн

80% исследования — CSS. 80% работы уйдёт на данные: CORS, 76K строк, пагинация Битрикс, кэширование.

4. Шесть разных палитр

Разные hex-коды в файлах 01, 03, 07. Нужна ОДНА палитра, записанная в одном месте.

5. Новогодняя ёлка из эффектов

14 эффектов одновременно. Stripe использует 2-3. Выбрать максимум 4-5.

6. "Must-have" = всё и ничего

25 виджетов помечены Must-have. Канбан = ~1000 строк кода. Таблица с фильтрами = ~500. Это отдельные приложения.

7. Референсы — продукты команд из 50+ человек

Stripe (1000+), Linear (70+), Vercel (500+). Полезнее смотреть на Tabler (1 разработчик).

8. Дублирование между файлами

KPI-карточка описана 3 раза. Палитра — 3 раза с разными hex. fadeInUp — 3 раза.

9. Нет ответа "откуда данные"

CORS, аутентификация, 76K строк = 5-15MB CSV. Нужен бэкенд? Кэш? Прокси?

10. Не actionable

Нет макета, нет спеки данных, нет плана итераций, нет архитектурного решения.

Что хорошо

  • Контент-карта (05) — отличная декомпозиция
  • Живые примеры (04) — 27 ссылок, можно потрогать
  • Антипаттерны (06) — список ошибок ценнее "лучших практик"
  • ApexCharts — правильный выбор
  • tabular-nums — мелочь, но критичная
  • Правило 5-7 KPI — важное ограничение

Правильный порядок

  1. Архитектура (30 мин): статический HTML или JS + API или бэкенд + фронт
  2. MVP (1 день): 5-7 KPI, 2-3 графика, 1 таблица, данные hardcoded
  3. Данные (2-3 дня): Google Sheets, потом Битрикс, кэш в localStorage
  4. Красота (1 день): 2-3 эффекта, проверка на реальном устройстве
  5. Итерации: добавлять секции по одной
Исследование собрало отличный материал, но предлагает построить космический корабль когда нужен хороший велосипед. Нужно урезать скоуп в 5 раз и начать с MVP.

09. Восприятие и удобство

Как 3 партнёра читают дашборд. Eye-tracking, когнитивная нагрузка, actionability.

Главный принцип

Дашборд отвечает на 3 вопроса: Где мы? (статус) | Где должны быть? (план) | Что требует внимания? (действия)

Тест 5 секунд

Партнёр открыл дашборд. Через 5 секунд закрыл. Должен запомнить:

  • Бизнес ОК или нет — общий RAG-статус
  • Ключевая цифра — North Star вверху слева
  • Тренд — стрелка или спарклайн

Eye-tracking (F-паттерн)

======================== <- 1. Горизонтально по верху ============ <- 2. Вниз по левому краю ================ <- 3. Ещё горизонтальное сканирование ====== <- 4. Внимание падает к низу

Вывод: North Star metric = верхний левый угол, крупный шрифт. Нижний правый = "мёртвая зона".

Когнитивная нагрузка: не 7, а 4

Итого: 4 блока x 2-3 метрики = 8-12 метрик на экране. Не 50.

North Star Metric

Рекомендация: Прибыль инвесторам нарастающим итогом (37 млн) — "пульс" бизнеса.

Actionability: Красное = Действие

Не так: Объект #47 — КРАСНЫЙ
А так: ! Объект #47 Арбат — задержка ремонта 12 дней -> Связаться с подрядчиком (отв: Иванов) -> При текущем темпе: -400К/мес

Мобильный вид

Мобильный = НЕ уменьшенный десктоп. Отдельный интерфейс: 3-4 карточки, 1 колонка, шрифт 24-32px.
