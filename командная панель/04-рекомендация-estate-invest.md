# Рекомендация: Командный центр Estate Invest

---

## Контекст

- ~10 сотрудников, 3 партнёра
- 118 ИП (объектов), 900+ инвесторов, ~2000 зарегистрированных
- ~196 млн покупка, ~388 млн продажа, ~37 млн прибыль
- 16 Google Sheets, 117 индивидуальных касс
- Битрикс24 CRM: 4 воронки, 4030 сделок

- Telegram: боты (Salebot, Денежный бот), каналы

- Уже есть наработки: Streamlit (data_loader.py, analytics.py, app.py)

## Три сценария реализации

### Сценарий A: Минимум усилий (для старта)

```
Данные:     Google Sheets (уже есть)
ETL:        n8n Cloud (уже есть) → агрегация в сводный лист
Фронтенд:  Yandex DataLens (бесплатно) или Looker Studio
Алерты:    Telegram (уже работает)
```

| Параметр | Значение |
|----------|----------|
| Стоимость | $0 дополнительно |
| Время настройки | 2-4 часа |
| Покрытие потребностей | ~70% |

**Шаги**:
1. Создать сводную Google-таблицу "DWH" с листами: Сделки, Финансы, Инвесторы, Объекты, Авито, Метрики
2. IMPORTRANGE из существующих 16 таблиц для ключевых данных
3. n8n workflow: Битрикс24 REST API → парсинг → запись в DWH (раз в час)
4. Подключить DataLens/Looker Studio к сводной таблице
5. Создать чарты: воронка, финансы, статус объектов, динамика инвесторов

**Плюсы**: ноль затрат, работает с текущей инфраструктурой, DataLens — российский.
**Минусы**: нет real-time, ограниченная кастомизация, медленно при >10К строк.

### Сценарий B: Баланс мощности и простоты (РЕКОМЕНДУЕТСЯ)

```
Данные:     PostgreSQL (Hetzner VPS)
Кэш:       Redis (тот же VPS)
ETL:        n8n Cloud → PostgreSQL
Фронтенд:  Grafana OSS (Docker)
Алерты:    Grafana Alerts + Telegram
```

```
┌─────────────────────────────────────────┐
│         Hetzner VPS CX22 ($5/мес)       │
│                                          │
│  ┌──────────┐  ┌───────┐  ┌──────────┐ │
│  │PostgreSQL│  │ Redis │  │ Grafana  │ │
│  │  :5432   │  │ :6379 │  │  :3000   │ │
│  └────┬─────┘  └───┬───┘  └────┬─────┘ │
│       └────────────┼──────────┘         │
│                    │                     │
│  ┌─────────────────▼──────────────────┐ │
│  │     Caddy (HTTPS reverse proxy)     │ │
│  │     dashboard.estateinvest.online   │ │
│  └────────────────────────────────────┘ │
└─────────────────────────────────────────┘
        ▲                    │
   n8n Cloud            Telegram
 (ETL, пишет           (алерты)
  в PostgreSQL)
```

| Параметр | Значение |
|----------|----------|
| Стоимость | ~$5/мес (VPS) |
| Время настройки | 1-2 дня |
| Покрытие потребностей | ~95% |

**Шаги**:
1. Арендовать Hetzner CX22 (2 vCPU, 4 GB RAM, €4.51/мес)
2. Docker Compose: PostgreSQL + Redis + Grafana + Caddy
3. Создать таблицы: objects, investors, deals, cash_flow, avito_stats
4. n8n workflows: каждый час → Битрикс24/Sheets → PostgreSQL
5. Grafana: дашборды с auto-refresh 30 сек, алерты в Telegram
6. Caddy: HTTPS на поддомене dashboard.estateinvest.online

**Плюсы**: настоящая БД, real-time, алерты, расширяемость, $5/мес.
**Минусы**: нужен VPS, начальная настройка Docker.

### Сценарий C: AI-assisted (быстрый кастомный UI)

```
Данные:     Google Sheets / Supabase
ETL:        n8n Cloud
Фронтенд:  Lovable → React dashboard → Vercel
Алерты:    Telegram
```

| Параметр | Значение |
|----------|----------|
| Стоимость | $0-20/мес |
| Время настройки | 3-5 дней |
| Покрытие потребностей | ~90% |

**Идея**: описать дашборд словами в Lovable/Bolt.new → получить React-приложение → деплой на Vercel бесплатно.

**Плюсы**: кастомный UI, быстрая разработка через AI, бесплатный деплой.
**Минусы**: зависимость от AI-платформы, нужна доработка.
