◆ Рыночная возможность

Проблема

Банкротные торги и аукционы госимущества -- один из самых доходных каналов покупки недвижимости. Средний дисконт к рыночной цене составляет 20-60%. Но работать с ними сложно:

  • Информация разбросана по 10+ площадкам (torgi.gov.ru, ЕФРСБ, Сбербанк-АСТ, lot-online, РАД и др.)
  • Нет оценки -- ни одна площадка не показывает рыночную стоимость лота
  • Устаревший UX -- государственные площадки выглядят на уровне 2005 года
  • Много мусора -- нужно вручную отфильтровывать тысячи неинтересных лотов

Решение

Автоматический скринер, который:

  1. Собирает лоты со всех площадок в одном месте
  2. Обогащает данными кадастра (площадь, стоимость, координаты)
  3. Оценивает рыночную стоимость по аналогам с ЦИАН, Авито, Домклик
  4. Считает реальный дисконт и показывает только выгодные лоты
  5. Присылает уведомления в Telegram

Незанятая ниша

Что есть у конкурентов

  • Агрегация лотов с площадок
  • Фильтры по региону и типу
  • Уведомления о новых лотах
  • Веб-интерфейс

Чего НЕТ ни у кого

  • AI-оценка рыночной стоимости
  • Автоматический расчёт дисконта
  • Telegram-first интерфейс
  • Интеграция с кадастром
В США аналог -- PropStream ($49-79/мес, 150M+ объектов). Начинался с парсинга одного штата. Сейчас лидер рынка. Telegram-first продукта с аналитикой не существует нигде в мире -- это белое пятно.

Алгоритм оценки (пошагово)

  1. Получаем адрес и кадастровый номер лота с аукциона
  2. DaData -- нормализация адреса, координаты, привязка к ФИАСФИАС -- Федеральная информационная адресная системаЕдиная государственная база всех адресов в России. Каждому зданию присвоен уникальный ID (GUID). Это как ИНН, но для адресов. Позволяет однозначно определить, что "ул. Ленина 15", "Ленина, д.15" и "пр. Ленина 15" -- это один и тот же дом. Критично для дедупликации объектов с разных площадок.
  3. Росреестр -- кадастровая стоимость, площадь, год постройки
  4. ЦИАН + Авито + Домклик -- находим 5-10 аналогов в радиусе 1-2 км
  5. Считаем медиану цены за кв.м. аналогов
  6. Применяем корректировкиКорректировкиПоправки к цене аналога, чтобы учесть различия. Например, если аналог -- кирпич, а наш объект -- панель, применяем скидку -5-10%. Если 1-й этаж вместо 5-го -- ещё -5%. Сумма всех поправок не должна превышать 15-20%, иначе объекты слишком разные для сравнения. (этаж, ремонт, тип дома)
  7. Результат: рыночная оценка + кадастровая + дисконт аукциона

Индикация выгодности

Более 40% -- отличная сделка 20-40% -- хорошая 10-20% -- средняя Менее 10% -- слабая Выше рынка -- невыгодная

Как оцениваем рыночную стоимость

1 Фаза 1 (MVP): Сравнительный подход без ML

Находим 5-10 аналогов в радиусе 1-2 км, считаем медиану цены за кв.м., применяем корректировкиЧто такое корректировки?Это поправки к цене аналога, чтобы учесть различия с нашим объектом. Если аналог на 5-м этаже, а наш на 1-м -- применяем скидку -5%, потому что первый этаж дешевле.Пример: Аналог стоит 4 000 000 руб (кирпич, 5 этаж, 55 кв.м.). Наш объект -- панель, 1 этаж, 50 кв.м.
Панель вместо кирпича: -7%
1-й этаж вместо 5-го: -5%
Площадь меньше: +5%
Итого корректировка: -7% = 3 720 000 руб по ФСО VФСО V -- Федеральный стандарт оценки №5Официальный российский стандарт оценки недвижимости. Обязателен для профессиональных оценщиков. Устанавливает правила: сколько нужно аналогов, как корректировать цены, какие ограничения. Мы используем упрощённую версию этого стандарта для автоматической оценки..

ФакторДиапазонКак определяем
Район+-10-30%Средняя цена за кв.м. по району
Этаж (1-й и последний)-3% до -7%Первый/последний этаж дешевле
Площадь+-5-15%Больше площадь = ниже цена за кв.м.
Состояние/ремонт+-5-20%Из описания лота / фото
Тип дома+-5-10%Кирпич > панель > блок
Год постройки+-3-10%Новые дома дороже

Минимум 3-5 аналоговПочему минимум 3-5?По ФСО V нужно число факторов + 1 аналогов. Если корректируем по 3 факторам (этаж, площадь, тип дома), нужно минимум 4 аналога. Это как система уравнений: чтобы решить 3 неизвестных, нужно минимум 4 уравнения. Иначе корректировки статистически не обоснованы.. Сумма всех корректировок не более 15-20%Лимит корректировок 15-20%Если приходится слишком сильно корректировать аналог -- значит, он слишком отличается от нашего объекта и не годится для сравнения.Пример: Аналог в другом районе (+15%), другой этаж (-5%), другой тип дома (-7%), площадь разная (+8%). Итого: 35% корректировок -- слишком много, аналог нужно заменить на более похожий.. Для земли и коммерции -- кадастровая стоимость x 1.3-1.5 как fallback.

2 Фаза 2: ML-модель (LightGBM) нужно 2000+ объектов
  • 20+ признаков: площадь, этаж, расстояние до центра, тип дома, год постройки, инфраструктура
  • Ожидаемая точность для Перми: MAPEMAPE -- Mean Absolute Percentage ErrorСредняя ошибка оценки в процентах. Если MAPE = 15%, значит наша оценка в среднем отличается от реальной цены на 15%. Для контекста: профессиональный оценщик ошибается на 10-15%, а Zillow -- на 2-7%. 15-25% на старте, цель -- довести до 10-15%
  • SHAPSHAP -- объяснение оценкиМетод, который показывает, какие факторы и насколько повлияли на итоговую цену. Например: "площадь 65 кв.м. добавила +800 000 руб, а первый этаж убавил -300 000 руб". Повышает доверие к оценке -- пользователь видит не просто цифру, а логику. для объяснения оценки ("почему такая цена")
3 Фаза 3: Ensemble + Computer Vision стратегия
  • Ансамбль моделей (LightGBM + XGBoost + CatBoost)
  • Оценка состояния по фото (нейросеть CLIP)
  • ARV-калькулятор (стоимость после ремонта)

► Применение к Estate Invest: MVP Ключевой раздел

Зачем это нам

Estate Invest уже работает с аукционной недвижимостью: 118 объектов, 900+ инвесторов, ~196 млн покупок, ~388 млн продаж. Сейчас поиск и оценка объектов -- ручной процесс. Автоматизация даст:

  • Скорость -- узнавать о выгодных лотах первыми, а не из ручного мониторинга
  • Объём -- обрабатывать 500+ лотов вместо десятков
  • Объективность -- оценка на данных, а не на интуиции
  • Масштабирование -- из внутреннего инструмента в продукт для инвесторов

Что делаем в MVP (Фаза 1)

1 Мониторинг торгов -- Telegram-бот 0 руб/мес
  • Парсим torgi.gov.ru (госимущество) + ЕФРСБ (банкротство) каждые 6 часов через REST API
  • Фильтруем: только недвижимость + Пермский край
  • Сохраняем в Google Sheets (до 10K строк)
  • Отправляем в Telegram новые лоты с базовой информацией
  • Простая оценка: дисконт к кадастровой стоимости (без ML)
  • Правило: дисконт > 30% к кадастровой = подсветить

Инструменты: n8n Cloud (уже есть) + Google Sheets + Telegram бот

Срок: 2-3 недели

  • Подключаем ads-api.ru -- рыночные цены с ЦИАН, Авито, Домклик
  • DaData -- координаты, нормализация адресов, кадастровые номера
  • Росреестр PKK -- кадастровая стоимость
  • Автоматический поиск аналогов в радиусе 1-2 км
  • Расчёт реального дисконта: цена торгов vs рыночная оценка
  • Telegram: полная карточка лота с оценкой и аналогами

⚠ Риски и ограничения

РискВлияниеМитигация
Мало данных в Перми среднее Пермь не Москва. 3000 чистых объектов -- оптимистичная цель. Начинаем с rule-based подхода, ML -- когда данные накопятся
n8n Cloud лимиты среднее 5 concurrent executions. Уже используем для других проектов. Тяжёлую работу выносим на FastAPI
Точность оценки на старте среднее MAPE 15-25% -- нормально для старта. У конкурентов оценки нет вообще. Показываем confidence рядом с ценой
ads-api.ru тестовый тариф низкое Тестовый тариф отдаёт price=0. Нельзя оценить до оплаты (2000 руб). Альтернатива: cianparser (бесплатно, но юр. риск)
Юридические риски парсинга низкое torgi.gov.ru = Open Data (минимальный). Для коммерческих площадок используем легальный API

Что точно работает

  • torgi.gov.ru имеет JSON API (подтверждено несколькими источниками, Open Data)
  • LightGBM -- proven подход для оценки недвижимости по всему миру
  • n8n Cloud + HTTP Request + Telegram -- уже используем для других проектов
  • DaData возвращает координаты и кадастровые данные
  • Overpass API (OpenStreetMap) бесплатно покрывает инфраструктуру Перми

Что нужно проверить перед стартом

  • torgi.gov.ru: точный catCode и структура ответа (живой HTTP-запрос)
  • DaData Suggestions: какие enriched-поля возвращает бесплатный эндпоинт
  • ads-api.ru: реальное качество данных на платном тарифе
