## 1. Описание проекта

### Что это

Система аналитики недвижимости, которая:
1. Скринит все лоты на торговых площадках (torgi.gov.ru, банкротные торги)
2. Оценивает рыночную стоимость каждого лота (квартиры, земля, коммерция)
3. Показывает дисконт к рынку

### Для кого

- **Estate Invest** — внутренний инструмент для поиска объектов
- **Инвесторы в недвижимость** — оценка выгодности аукционных лотов

### MVP scope

- Регион: **только Пермь**
- Источники: torgi.gov.ru + ЕФРСБ
- Оценка: сравнение с Авито/ЦИАН
- Интерфейс: Telegram-бот + Google Sheets
- **НЕ будет**: сайта, визуального дашборда, всей России

### torgi.gov.ru — главный источник (Open Data)

```
# Поиск лотов
GET https://torgi.gov.ru/new/api/public/lotcards/search
  ?catCode=2              # недвижимость
  &auctionType=BANKRUPT   # банкротство
  &dynSubjRF=59           # Пермский край
  &size=100&page=0
  &sort=firstVersionPublicationDate,desc
  &lotStatus=PUBLISHED,APPLICATIONS_SUBMISSION

# Карточка лота
GET https://torgi.gov.ru/new/api/public/lotcards/{lotId}

# Экспорт в Excel
GET https://torgi.gov.ru/new/api/public/lotcards/export/excel?...
```

**Данные карточки**: lotName, lotDescription, startPrice, depositAmount, biddStartTime/EndTime, auctionStartDate, characteristics[] (адрес, площадь, кадастровый номер), lotStatus, etpUrl, biddType, attachments[].

**Rate limit**: ~60-120 req/min, задержка 1-3 сек достаточно. CAPTCHA нет.

**SSL**: используется российский CA (не международный). Решение: `verify=False` или установить сертификат Минцифры.

**Лимит**: макс 10 000 результатов на запрос (100 стр x 100). Обход — разбивка по датам.

**Лицензия**: свободное коммерческое использование (Open Data).

### PKK Росреестра — кадастровые данные

```
GET https://pkk.rosreestr.ru/api/features/5/{кадастровый_номер}
```

- Возвращает: адрес, площадь, кадастровая стоимость, координаты
- Rate limit агрессивный (~10-20 req/min), кешировать
- Python-библиотеки: `rosreestr-api`, `rosreestr2coord`
- MCP-сервер: `mcp-rosreestr` (кадастр + права + обременения + GeoJSON)

**Кадастровая стоимость ниже рыночной на 20-40%**. Множитель 1.3-1.5x — грубая аппроксимация рыночной.

### DaData.ru — обогащение данных

- 10 000 запросов/день бесплатно, 30 req/сек
- Геокодинг: координаты по адресу (покрытие Пермь ~47%)
- Кадастровый номер по адресу и наоборот (0.20 руб/адрес)
- Стоимость кв.м. (тариф "Максимальный")

### ads-api.ru — единый API для рыночных данных

ЦИАН + Авито + Домклик + Яндекс + sob.ru в одном API:
- 1 000 руб/мес (базовый) → 3 000 руб/мес (полный с контактами)
- 14 дней тестовый доступ (макс 50 объявлений за запрос)
- Поля: цена, адрес, площадь, комнаты, этаж, координаты, фото, описание
- Пауза 5 сек между запросами
- **Для MVP — самый быстрый путь** (снимает юр. риски + все площадки)

### cianparser — бесплатная альтернатива

- `pip install cianparser` (Python 3.8+, обновлён окт. 2025)
- Парсит: цена, адрес, площадь (общ/жил/кухня), этаж, координаты, тип дома, год
- **НЕ парсит**: фото, описание
- С `with_extra_data=True` — в 5-10x медленнее
- Парсинг ~10K объявлений по Перми: 2-6 часов
- Защита ЦИАН: rate limiting (не Cloudflare), пауза 2-5 сек

### Юридические риски

| Источник | Риск | Почему |
|----------|------|--------|
| torgi.gov.ru | **Минимальный** | Open Data, обязаны быть общедоступными по закону |
| ЕФРСБ | **Низкий** | Публичная информация, но нет явной лицензии |

### Почему n8n НЕ подходит как основной инструмент

- 5000+ листингов — медленно
- Нет Playwright, PostGIS, ML
- Лимит 5 concurrent executions

### Рекомендуемый стек

```
Сбор данных:  Python (Scrapy + Playwright) → cron
Хранилище:    Google Sheets (MVP) → PostgreSQL + PostGIS (v2)
Оценка:       Python (pandas, scikit-learn, XGBoost)
Интерфейс:    Telegram бот (MVP) → Web dashboard (v2)
Оркестрация:  n8n (расписание, уведомления, Telegram)
Геокодинг:    DaData API (10K бесплатных запросов/день)
```

**Гибрид — правильный путь**: n8n как оркестратор, Python-бэкенд для тяжёлой работы.

### Стоимость MVP

~$25-60/мес (VPS + DaData + хостинг)

## 5. Алгоритм оценки

### Дисконты на торгах (реальные данные)

| Этап торгов | Дисконт к рынку |
|-------------|----------------|
| Первичные (аукцион на повышение) | 0% (может дороже) |
| Повторные (-10% от начальной) | ~10% |
| Публичное предложение (начало) | 10-20% |
| Публичное предложение (середина) | 30-50% |
| Публичное предложение (глубокое) | 50-75% |
| **Среднее по рынку** | **20-60%** |

### Фаза 1 (MVP): Rule-based comparable sales

**Алгоритм оценки лота (пошагово):**
```
1. Получить адрес/кадастровый номер лота с аукциона
2. DaData → координаты, кадастровый номер
3. Росреестр → кадастровая стоимость, площадь, год постройки
4. ads-api.ru / cianparser → 5-10 аналогов в радиусе 1-2 км
5. Рассчитать медиану цены за м2 аналогов
6. Применить корректировки
7. Результат: рыночная оценка + кадастровая + дисконт аукциона
```

**Корректировки (по ФСО V):**

| Фактор | Диапазон | Как определить |
|--------|----------|----------------|
| Местоположение (район) | ±10-30% | Средняя цена за м2 по району |
| Этаж (1-й и последний) | -3% до -7% | Первый/последний этаж дешевле |
| Площадь | ±5-15% | Больше площадь = ниже цена за м2 |
| Состояние/ремонт | ±5-20% | Из описания/фото |
| Тип дома | ±5-10% | Кирпич > панель > блок |
| Год постройки | ±3-10% | Новые дома дороже |

**Ограничение**: сумма ВСЕХ корректировок ≤ 15-20% (иначе аналог не подходит).

**Минимум аналогов**: 3-5 (по ФСО V = число факторов + 1).

**Confidence** = количество аналогов: < 3 низкий, 3-5 средний, > 5 высокий.

**Земля и коммерция** (мало аналогов): кадастровая стоимость × 1.3-1.5 как fallback.

### Фаза 2: ML-модель (XGBoost/LightGBM)

- Нужно 2000+ объектов с признаками
- 20+ фич: площадь, этаж, расстояние до центра, тип дома...
- SHAP для объяснения оценки

### Фаза 3: Ensemble + Computer Vision

- Фото для оценки состояния (CLIP)
- Ensemble моделей
- ARV-калькулятор (стоимость после ремонта)

### Мировые лидеры AVM (для ориентира)

| Платформа | Модель | Точность |
|-----------|--------|----------|
| Zillow Zestimate | Neural network | 1.83% on-market / 7.01% off |
| Redfin Estimate | ML + MLS | 1.98% / 7.72% |
| HouseCanary | ML + CV | 2.7% / 7.5% |
| Домклик (Сбер) | LightGBM | не раскрывают |

**Ключевой инсайт**: LightGBM/XGBoost доминируют в production AVM.

### Инструменты

- **OpenAVMKit** — `pip install openavmkit` — FOSS AVM (LightGBM/XGBoost/CatBoost)
- **Supabase** — PostgreSQL + PostGIS + API бесплатно до 500MB
- **H3** — гексагональная сетка Uber для price heatmaps
- **SHAP** — объяснение оценок ML-модели

## 8. Зарубежный опыт

### PropertyRadar — эталонный кейс

Основатель Sean O'Toole, запуск 2007. Начал с парсинга trustee sales **только в Калифорнии**. Первые 2 года — убытки ($50K/мес). Появление на 60 Minutes (CBS) дало взрывной рост. Сейчас: 150M+ properties, $49-79/мес.

**Паттерн**: один штат → один тип данных → расширение.

### Мировые лидеры прозрачности

| Страна | Система | Что доступно |
|--------|---------|-------------|
| **UK** | Land Registry | 20M+ реальных цен сделок с 1995, бесплатно |
| **Франция** | DVF | ВСЕ сделки открыто (CSV, API) с 2019 |
| **Германия** | GREIX | 2M+ транзакций с 1960 года, бесплатно |
| **Австралия** | CoreLogic AU | 10M+ объектов, 40 лет истории |
| **Швеция** | Hemnet | 90% рынка, финальные цены публичны |

### Что внедрить

1. Confidence scoring для каждой оценки (как PriceHubble/Hometrack)
2. Clearance rate аукционов (preliminary → final, как REIV в Австралии)
3. SHAP-объяснение оценки (какие факторы повлияли)

### Вариант A: n8n + Sheets (быстрый старт, 2 недели)

1. n8n workflow: парсинг torgi.gov.ru JSON API → Google Sheets

5. Price gap: ручные ссылки на аналоги ЦИАН/Авито
6. Дедупликация лотов

### Вариант B: Python + PostgreSQL (правильный путь, 4-8 недель)

| Неделя | Задача | Результат |
|--------|--------|-----------|
| 1-2 | Парсер torgi.gov.ru + PostgreSQL | База лотов Пермь |
| 2-3 | Парсер Авито/ЦИАН (через ads-api.ru) | Рыночные цены |
| 3-4 | Rule-based оценка + Telegram-бот | Уведомления + оценка |
| 4-5 | PKK интеграция + price history | Кадастр + тренды |
| 5-6 | Мониторинг торгов + price gap | Дисконт к рынку |
| 6-8 | XGBoost модель (когда 2000+ объектов) | ML-оценка |

### Что работало у других / что нет

| Работало | Не работало |
|----------|------------|
| Мониторинг + уведомления (killer feature) | Охватить все площадки сразу |
| Сравнение с рынком (price gap) | Автоматическая оценка на старте |
| Фокус на одной категории | Широкий ассортимент без глубины |
| Бесплатный поиск + платная аналитика | Полностью платный с первого дня |
