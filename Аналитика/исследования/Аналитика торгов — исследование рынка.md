## 1. Кейсы существующих продуктов

### 1.1 Российские агрегаторы и аналитические платформы

#### TBankrot.ru — лидер рынка
- **Что делает**: самая полная база торгов по банкротству в РФ, ~90 000+ открытых лотов, ~3 000 новых ежедневно
- **Аналитика**: прогноз выигрыша на торгах, анализ организаторов/победителей, процент снижения цены, региональная конкуренция
- **AI**: автоматическая проверка карточек лотов на ошибки, рекомендации оптимальной ставки
- **Интерфейс**: веб + мобильное приложение (iOS/Android) + Telegram-канал @tbankrot_official
- **Тарифы**:
  - Бизнес: 1 500 руб/мес (12 600 руб/год)
  - Профи: 3 500 руб/мес (25 800 руб/год) — аналитика победителей, АУ/ОТ, heatmaps
  - Премиум: 5 800 руб/мес (48 600 руб/год) — полная аналитика, comparables по недвижимости
- **Пробный период**: 7 дней бесплатно (тариф Бизнес)

#### Bankrot-Spy.ru (Банкротный шпион)
- **Что делает**: поиск по 52 площадкам, категоризация (недвижимость, авто, оборудование, дебиторка и т.д.)
- **Ключевая фича**: сравнение цены на торгах с рыночной ценой (comparative valuation)
- **Доп. услуги**: аукционный робот для автоматических ставок на ЭТП
- **Аналитика**: реестры должников, организаторов, рейтинг победителей, топ лотов
- **Тарифы**: Basic (поиск + 2 рассылки + export) и VIP (+ оценка рыночной стоимости + без рекламы)

#### EI.ru (Единый Информатор)
- **Что делает**: маркетплейс электронных торгов, 1 000 000+ лотов с 55 площадок
- **Обновление**: 5 000+ новых лотов ежедневно
- **Доп. данные**: подтягивает данные из Росреестра и ГИБДД к лотам
- **Избранное**: организация лотов в папки, уведомления по сохраненным поискам

### 1.3 Американские аналоги (для понимания потолка рынка)

#### PropertyRadar (ex-ForeclosureRadar) — эталонный кейс
- **Основатель**: Sean O'Toole, запуск 2007
- **История**: начал с отслеживания trustee sales в Калифорнии. Первые 2 года — убытки ($50K/мес), затем появление на 60 Minutes (CBS) дало взрывной рост
- **MVP**: парсинг данных аукционов + веб-интерфейс для отслеживания
- **Сейчас**: 150+ million properties, 200+ фильтров, heatmaps, direct mail/email marketing встроенный
- **Тарифы**: Essential $49/мес, Complete $79/мес, Custom — по запросу
- **Ключевой инсайт**: начали с одного штата (California), одного типа данных (foreclosure auctions), потом расширились

#### Auction.com
- **Что делает**: крупнейшая онлайн-платформа аукционов REO/foreclosure в США
- **Модель**: buyer's premium 5-10% от цены сделки

#### Hubzu (Altisource)
- **Revenue**: $144.5M (2022), целевой уровень $247M при возврате к норме foreclosures
- **Модель**: listing fees + buyer's premium + analytics add-ons

#### DealMachine
- **Что делает**: 700 фильтров, AI Vision Builder для идентификации объектов
- **Ключевая фича**: driving-for-dollars (поиск объектов в поле через телефон)

## 2. MVP-подход: как начинали другие

### 2.1 Паттерн PropertyRadar (ForeclosureRadar)

**Фаза 1 — Один штат, один тип данных (6 мес)**:
- Парсинг данных trustee sales только в Калифорнии
- Простой веб-интерфейс для поиска и фильтрации
- Ключевая ценность: отслеживание postponements/cancellations аукционов (чего не было нигде)
- Первые клиенты — профессиональные flippers

**Фаза 2 — Расширение данных (12 мес)**:
- Добавление opening bids, price history
- Отслеживание 150+ trustees
- California Foreclosures Report → медиа-внимание → рост

**Фаза 3 — Nationwide + маркетинг (24+ мес)**:
- 150M+ объектов по всей стране
- Встроенный direct mail, email, online ads
- Ребрендинг в PropertyRadar

### 2.2 Паттерн TBankrot (ключевой российский кейс)

**Фаза 1**: агрегация лотов с основных ЭТП → единый поиск
**Фаза 2**: мониторинг + email-уведомления по фильтрам
**Фаза 3**: аналитика (победители, АУ/ОТ, ценовые тренды)
**Фаза 4**: AI-проверка карточек лотов + мобильное приложение

### 2.3 Что собирали в первую очередь

1. **Базовые данные лотов**: название, цена, категория, площадка, даты торгов
2. **Связь с источником**: URL на площадку, номер дела, данные должника
3. **Исторические цены**: начальная цена → снижение → финальная цена продажи
4. **Геолокация**: регион, город, адрес (для недвижимости)

### 2.4 Первичная оценка стоимости

- **TBankrot**: сравнение с историей продаж аналогичных лотов на торгах
- **Bankrot-Spy**: сопоставление цены лота с рыночной ценой (Avito, CIAN, Domclick)
- **PropertyRadar**: comparables по району + транзакционная история
- **Общий паттерн**: на старте — ручная/полуавтоматическая оценка, затем ML-модели

### 2.5 Что работало / что нет

| Работало | Не работало |
|----------|------------|
| Мониторинг + уведомления (killer feature #1) | Попытка сразу охватить все площадки |
| Сравнение с рынком (price gap analysis) | Полностью автоматическая оценка на старте |
| Фокус на одной категории (недвижимость) | Широкий ассортимент без глубины |
| Email/Telegram рассылки | Только веб без мобильного |
| Бесплатный поиск + платная аналитика | Полностью платный доступ с первого дня |

- **Bankrot-Spy VIP**: включает рыночную оценку авто и недвижимости
- **TBankrot Премиум**: comparables, конкурентный анализ по лоту

**Бесплатно**: базовый поиск, просмотр лотов, ограниченные фильтры
**Платно**: мониторинг, рассылки, аналитика, экспорт, оценка стоимости

## 4. Юридические аспекты (Россия)

### 4.1 Парсинг torgi.gov.ru — ЛЕГАЛЬНО

**torgi.gov.ru предоставляет Open Data**:
- Официальный раздел открытых данных: `torgi.gov.ru/opendata/`
- Формат: XML с параметрами через URL
- URL-шаблон: `torgi.gov.ru/opendata/7710349494-torgi/data.xml?bidKind={bidKind}&publishDateFrom={...}&publishDateTo={...}`
- **Условия использования**: можно свободно, бесплатно, бессрочно и без территориальных ограничений использовать (включая многократное использование), в том числе право копировать, публиковать, распространять, изменять и комбинировать с другой информацией, использовать как для некоммерческих, так и для коммерческих целей
- **Источник**: Минэкономразвития России

**Вывод**: torgi.gov.ru — один из самых юридически безопасных источников для парсинга в России. Данные явно открыты для коммерческого использования.

### 4.2 ЕФРСБ (bankrot.fedresurs.ru) — условно легально

- Данные публичны и доступны для просмотра бесплатно
- Сторонние API (api-assist.com, you-right.ru) уже предоставляют доступ — 200 запросов/мес бесплатно
- Обновления в реальном времени
- **Риск**: нет явной open data лицензии как у torgi.gov.ru, но данные де-факто публичны

### 4.3 Парсинг Авито/ЦИАН — ВЫСОКИЙ РИСК

**Прецеденты**:
- **Avito vs ЦИАН (2017-2018)**: ФАС выдала предупреждение ЦИАН за копирование 150 000+ объявлений. ЦИАН удалил ~140 000 объявлений. Основание — недобросовестная конкуренция
- **VKontakte vs Double (2021)**: суд признал извлечение пользовательских данных нарушением прав на базу данных (ст. 1335.1 ГК РФ), даже если данные публичны
- **Avito vs Auto.ru (Яндекс)**: нижестоящие суды отклонили иск, т.к. 300 000+ объявлений якобы были размещены продавцами добровольно на обеих площадках

**Выводы по Авито/ЦИАН**:
- Массовое копирование контента (описания, фото) — нарушение
- Извлечение фактических данных (цены, площади) — серая зона
- Обход технических защит (CAPTCHA, rate limiting) — может квалифицироваться по ст. 272 УК РФ
- **Рекомендация**: для оценки рыночной стоимости использовать Avito API (если доступен) или ручной/полуавтоматический сбор comparables

### 4.4 Кадастровые данные Росреестра — ОГРАНИЧЕНИЯ

- Публичная кадастровая карта (pkk.rosreestr.ru) — данные доступны для просмотра
- **Коммерческое использование**: запрещено без договора с Росреестром
- API-сервисы (Контур.Реестро, ktotam.pro, api-cloud.ru) предоставляют легальный доступ за плату
- **Рекомендация**: использовать лицензированные API вместо прямого парсинга

### 4.5 Персональные данные — ЗАПРЕЩЕНО

**С 1 марта 2021 (ФЗ-519)**:
- Массовый автоматический сбор персональных данных из интернета запрещён, даже если данные "публичны"
- Понятие "общедоступные персональные данные" больше не является основанием для обработки
- ФИО, телефоны, email, адреса в описаниях лотов — это персональные данные
- **Рекомендация**: при парсинге описаний лотов — автоматически удалять/анонимизировать ПД

### 4.6 Итоговая матрица рисков

| Источник | Риск | Коммерческое использование | Рекомендация |
|----------|------|---------------------------|--------------|
| torgi.gov.ru | Минимальный | Явно разрешено (Open Data) | Основной источник |
| ЕФРСБ | Низкий | Де-факто публичные данные | Использовать с осторожностью |

| Авито | Высокий | Запрещено без согласия | Только для ручных comparables |
| ЦИАН | Высокий | Запрещено без согласия | Только для ручных comparables |
| Росреестр | Средний | Через лицензированные API | Использовать Контур.Реестро и аналоги |
| Domclick/Яндекс.Недвижимость | Высокий | Запрещено без API-договора | Избегать прямого парсинга |

### 5.2 Telegram-бот vs веб-приложение

**Telegram-бот как MVP-интерфейс — однозначный выбор**:

Преимущества:
- Нулевая стоимость привлечения (пользователи уже в Telegram)
- Push-уведомления "из коробки"
- Нет необходимости в хостинге фронтенда
- Inline-кнопки для фильтрации
- Мгновенный time-to-market
- Совместим с n8n (HTTP Request → Telegram API)

Ограничения:
- Сложные фильтры неудобны в чат-интерфейсе
- Нет визуализации (карты, графики)
- Нет сохранённых поисков (нужно через staticData или БД)

**Рекомендация**: Telegram-бот для MVP (первые 3-6 мес), параллельно собирать данные для веб-интерфейса

### 5.3 Google Sheets как база данных для MVP

**Плюсы**:
- Уже используется в проекте СКАУТ для Avito
- Бесплатный хостинг данных
- Ручной просмотр/редактирование
- Нативная интеграция с n8n
- Формулы для простой аналитики

**Минусы**:
- Лимит 10M ячеек / лист (достаточно для MVP)
- Медленные запросы при большом объёме
- Нет нормальных JOIN/индексов

**Рекомендация**: Google Sheets для первых 1000-5000 лотов, затем миграция на Supabase/PostgreSQL

### 5.4 Ручная vs автоматическая оценка

**Фаза 1 (MVP)**: ручная оценка
- Пользователь видит лот → вручную ищет аналоги на Avito/CIAN/Domclick
- Бот предоставляет удобные ссылки для быстрого поиска аналогов
- **Ценность**: экономия времени на поиск лота, а не на оценку

**Фаза 2**: полуавтоматическая
- Автоматический подбор аналогов по параметрам (район, площадь, тип)
- Показ средней рыночной цены из собственной базы
- **Реализация**: агрегация цен из собственных парсингов Avito (если уже есть)

**Фаза 3**: автоматическая
- ML-модель оценки (gradient boosting / neural network)
- Требует: 10 000+ исторических продаж с ценами

### 5.5 Рекомендуемый стек для MVP

```
Источник данных:
  torgi.gov.ru Open Data (XML) → n8n HTTP Request → парсинг

Хранилище:
  Google Sheets (MVP) → Supabase (масштабирование)

Автоматизация:
  n8n (schedule trigger → парсинг → фильтрация → уведомление)

Интерфейс:
  Telegram Bot (через n8n HTTP Request к Telegram API)

Оценка:
  Ручная (фаза 1) → полуавтоматическая с comparables (фаза 2)
```

5. Price gap: автоматическое сравнение с историческими ценами из Sheets
6. Дедупликация лотов (по ID объявления)

**Результат**: работающий Telegram-бот, который ежедневно присылает новые лоты банкротных торгов с базовой аналитикой (дисконт к начальной цене, тренд снижения).

---

## 6. Выводы и рекомендации

### 6.1 Рынок

- Рынок агрегаторов банкротных торгов в России **зрелый** — 10+ активных сервисов
- Но ниша **аналитики и оценки** — **слабо развита**. TBankrot единственный с серьёзной аналитикой
- Telegram-first продукта с AI-аналитикой **не существует** — это белое пятно
- Потолок рынка (по аналогии с USA): PropertyRadar оценивается в десятки $M revenue

### 6.2 Дифференциация

Чтобы отличаться от существующих агрегаторов:
1. **AI-оценка рыночной стоимости** — ни один российский сервис не делает это хорошо
2. **Telegram-first UX** — все конкуренты web-first
3. **Фокус на одной категории** — недвижимость (как PropertyRadar начинал с foreclosures)
4. **Интеграция с кадастровыми данными** — автоматическое обогащение лотов данными Росреестра

### 6.3 Монетизация (рекомендация)

**Фаза 1 (0-6 мес)**: бесплатный бот → набор аудитории
**Фаза 2 (6-12 мес)**: freemium — бесплатные уведомления, платная аналитика (1 500 руб/мес)
**Фаза 3 (12+ мес)**: отчёты по лотам (500-2 000 руб), white-label API, агентская модель

### 6.4 Юридический путь наименьшего сопротивления

1. torgi.gov.ru Open Data — **основной безопасный источник**
2. ЕФРСБ — дополнительная информация о процедурах банкротства
3. Росреестр — через лицензированные API (Контур.Реестро)
4. Авито/ЦИАН — **только ручные comparables**, никакого автоматического парсинга
5. Собственная база цен — копить данные о продажах из открытых источников
