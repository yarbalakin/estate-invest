## 1. Архитектура

```
ИСТОЧНИКИ ДАННЫХ                ПАРСЕРЫ (cron */4h)              ХРАНИЛИЩЕ
─────────────────               ───────────────────              ─────────
torgi.gov.ru ──────────→ torgi_monitor.py ───────┐
                                                  ├──→ Supabase (properties)
ЕФРСБ (bankrot.fedresurs.ru) ──→ efrsb_monitor.py ┘          │
                                                               │
TBankrot (агрегатор) ──────────→ tbankrot_scraper.py ──────────┘
                                                               │
                        ЦЕПОЧКА ОБОГАЩЕНИЯ                     │
                        ──────────────────                     │
                        enrich_cadastral.py ←──────────────────┤
                          (НСПД API)                           │
                              │                                │
                        enrich_market.py ←─────────────────────┤
                          (ads-api.ru)                         │
                              │                                │
                        enrich_scoring.py ←────────────────────┘

                              │
                              ▼
                        lots-monitor.html
                        (фронтенд: карточки, таблица, Яндекс Карты)
```

Все скрипты запускаются на VPS (5.42.102.36), данные хранятся в Supabase (PostgreSQL).

## 3. Обогащение

### 3.1. enrich_cadastral.py — Кадастровые данные

**Источник:** НСПД API (`nspd.gov.ru/api/geoportal/v2/search/geoportal`)

**Что получает:**
- Координаты (lat, lon)
- Кадастровая стоимость (cadastral_value)
- Площадь по кадастру (cadastral_area)
- Категория земли (cadastral_category)
- Разрешенное использование (cadastral_permitted_use)
- Обременения (encumbrance_type)
- Полигон границ (boundary_geojson) — для участков
- Этаж квартиры (apartment.floor)
- Соотношение цена/кадастр (cadastral_price_ratio)

**Запуск:** `python enrich_cadastral.py` на VPS

---

### 3.2. enrich_market.py — Рыночная оценка

**Источник:** ads-api.ru (агрегатор объявлений)

**Алгоритм:**
1. Определение `property_type` через `resolve_property_type()` — анализ названия и категории лота (приоритет: название > категория > БД)
2. Запрос аналогов через ads-api.ru по координатам, типу и площади
3. Фильтрация Haversine (по радиусу от объекта)
4. Фильтр площади +-50% (если мало аналогов — повтор без фильтра)
5. Отсев несовместимых категорий (земля не сравнивается с квартирами)
6. **IQR очистка выбросов**: Q1 - 1.5*IQR ... Q3 + 1.5*IQR
7. Медиана цен за единицу площади
8. **Торговая корректировка x0.9** (листинговые цены завышены на 5-15%)

**Источники объявлений (через ads-api.ru):** Авито, ЦИАН, Яндекс.Недвижимость, Домклик.

**Радиусы Haversine (город, село):**

| Тип | Город | Село |
|-----|-------|------|
| apartment | 1500 м | 1500 м |
| commercial | 2000 м | 2000 м |
| land | 2500 м | 10000 м |
| house | 3000 м | 5000 м |

**Маппинг property_type -> ads-api category_id:**

| property_type | ads category_id |
|---------------|----------------|
| apartment | 2 |
| house | 4 |
| land | 5 |
| commercial | 7 |

**Доходный метод (коммерция):**
Для коммерческой недвижимости дополнительно рассчитывается оценка через аренду:
- Медиана арендных ставок (руб/м2/мес) с IQR-очисткой
- Полезная площадь = площадь x 0.8
- Годовой доход = медиана x полезная площадь x 12
- Рыночная цена = годовой доход x cap_rate_years

**Cap rate (лет окупаемости):**

| Тип объекта | Лет |
|-------------|-----|
| Стрит-ритейл / торговое | 8 |
| Склад | 6 |
| Офис | 12 |
| Гараж / бокс | 10 |
| По умолчанию | 10 |

**Кадастровый фаллбек:**
Если нет аналогов — `market_price = cadastral_value x multiplier`:

| Тип | Множитель |
|-----|-----------|
| apartment | 1.2 |
| house | 1.3 |
| land | 1.3 |
| commercial | 1.5 |

### 3.3. enrich_scoring.py — Скоринг

Рассчитывает 3 балла (0-100), рекомендацию и финансовые оценки. Подробности в разделах 4-7.

---

## 4. Скоринг invest_score (0-100)

Инвестиционная привлекательность. Чем выше — тем лучше.

### Дисконт к рынку (0-60 баллов) — главный фактор

| Дисконт | Баллы |
|---------|-------|
| >= 50% | 60 |
| >= 40% | 50 |
| >= 30% | 40 |
| >= 20% | 28 |
| >= 10% | 15 |
| < 10% | 5 |
| Нет рыночной оценки | 5 |

### Качество оценки (0-10 баллов)

| Условие | Баллы |
|---------|-------|
| confidence = "высокая" И аналогов >= 10 | 10 |
| confidence = "высокая" или "средняя" И аналогов >= 3 | 6 |
| аналогов >= 1 | 3 |
| confidence = "кадастр" (фаллбек) | 0 |

### Кадастровое подтверждение (0-10 баллов)

`cadastral_price_ratio = цена лота / кадастровая стоимость`

| Ratio | Баллы | Значение |
|-------|-------|----------|
| < 0.7 | 10 | Цена значительно ниже кадастровой |
| < 1.0 | 6 | Цена ниже кадастровой |
| < 1.5 | 3 | Цена умеренно выше кадастровой |
| >= 1.5 | 0 | — |

### Тип объекта (0-5 баллов)

| Тип | Баллы |
|-----|-------|
| apartment (квартира) | 5 |
| house (дом) / commercial (коммерция) | 3 |
| land (земля) | 1 |

### Локация (0-5 баллов)

| Условие | Баллы |
|---------|-------|
| Известен район (district) | 3 |
| Есть координаты (lat, lon) | 2 |

### Тип торгов (0-5 баллов)

| Тип | Баллы |
|-----|-------|
| Реализация имущества должников (арест) | 5 |
| Приватизация | 3 |
| Прочее | 0 |

### Штраф

| Условие | Эффект |
|---------|--------|
| Цена < 50 000 руб | -15 баллов (скорее всего доля или ошибка) |

**Максимум:** 100 баллов. **Формула:** сумма всех компонентов, clamp 0-100.

## 5. risk_score (0-100)

Уровень риска. Чем выше — тем рискованнее. Базовый уровень: 20.

| Фактор | Баллы | Условие |
|--------|-------|---------|
| Базовый риск | +20 | Всегда |
| Нет координат | +15 | lat или lon отсутствуют |
| Нет кадастрового номера | +20 | cadastral_number пуст или < 5 символов |
| Нет рыночной оценки | +20 | market_price = null |
| Мало аналогов | +10 | analogs_count < 3 (при наличии market_price) |
| Очень низкая цена | +25 | 0 < price < 50 000 руб |
| Обременения | +15 | has_encumbrances = true |
| Долевая собственность (обременение) | +10 | encumbrance_type содержит "долевая" |
| Доля в праве (название) | +15 | name содержит "доля" или "доли" |
| Земля без района | +10 | property_type = "land" И district пуст |

**Максимум:** 100. **Формула:** 20 + сумма применимых факторов, clamp 0-100.

## 6. liquidity_score (0-100)

Скорость перепродажи. Чем выше — тем быстрее продастся. Базовый уровень: 30.

### Тип объекта (0-30 баллов)

| Тип | Баллы |
|-----|-------|
| apartment | +30 |
| house | +15 |
| commercial | +10 |
| land | +5 |

### Площадь квартиры (0-15 баллов, только apartment)

| Площадь | Баллы |
|---------|-------|
| 25-90 м2 | +15 |
| 91-150 м2 | +8 |
| > 150 м2 | +3 |
| < 25 м2 | +5 |

### Ценовой сегмент (0-15 баллов)

| Цена | Баллы |
|------|-------|
| <= 2 000 000 руб | +15 |
| <= 5 000 000 руб | +10 |
| <= 10 000 000 руб | +5 |
| > 10 000 000 руб | 0 |

### Локация

| Условие | Баллы |
|---------|-------|
| Известен район | +10 |

### Штрафы

| Условие | Эффект |
|---------|--------|
| Доля в праве (name содержит "доля"/"доли") | -20 |

**Максимум:** 100. **Формула:** 30 + сумма компонентов - штрафы, clamp 0-100.

## 7. Рекомендации (make_recommendation)

Текстовая рекомендация на основе трёх оценок:

| Условие | Рекомендация |
|---------|-------------|
| Нет market_price | "Нет рыночной оценки — требует ручной проверки" |
| invest >= 65, risk <= 35 | "Отличная сделка — высокий дисконт, низкий риск" |
| invest >= 65, risk <= 55 | "Хороший вариант — проверить документы" |
| invest >= 50, risk <= 45 | "Интересный объект — стоит изучить" |
| risk >= 65 | "Высокий риск — требует тщательной проверки" |
| invest < 30 ИЛИ discount < 10 | "Слабый дисконт — не рекомендуется" |
| Прочее | "Средний вариант — анализировать дополнительно" |

### Финансовые оценки (calc_estimated)

**Прибыль:**
```
estimated_profit = market_price - price - (market_price * 0.12)
estimated_roi = estimated_profit / price * 100
```
Транзакционные расходы 12% = налог (13%) + регистрация + агент.

**Срок продажи (estimated_sell_time, месяцев):**

| Тип | Цена <= X млн | Месяцев | Цена > X млн | Месяцев |
|-----|---------------|---------|---------------|---------|
| apartment | <= 3 | 2 | <= 6: 4, <= 12: 6 | > 12: 10 |
| house | <= 5 | 6 | > 5 | 12 |
| commercial | <= 5 | 6 | > 5 | 18 |
| land | <= 2 | 12 | > 2 | 24 |
| прочее | — | 12 | — | 12 |

## 8. Уровни confidence

Оценка надёжности рыночной цены:

| Уровень | Условие | Значение |
|---------|---------|----------|
| высокая | >= 10 аналогов И радиус < 2 км | Много близких аналогов, оценка надёжная |
| средняя | >= 3 аналогов | Достаточно аналогов для оценки |
| низкая | < 3 аналогов (но есть хотя бы 2) | Мало данных, оценка приблизительная |
| кадастр | 0 аналогов, использован кадастровый фаллбек | `cadastral_value * multiplier`, ненадёжно |

## 9. ads-api.ru — Источник рыночных данных

**Что это:** Платный API-агрегатор объявлений о продаже/аренде недвижимости в России.

**Откуда берёт данные:**
- Авито
- ЦИАН
- Яндекс.Недвижимость
- Домклик

**Ограничения:**
- Это **листинговые цены** (цены предложения), НЕ цены реальных сделок
- Объявления могут быть уже проданными, но ещё не снятыми с публикации
- Продавцы часто завышают цены — поэтому применяется **торговая корректировка x0.9**
- Задержка API: 5 секунд между запросами (`ADS_API_DELAY = 5`)

**Фильтрация аналогов:**
1. По радиусу (Haversine distance от объекта)
2. По площади +-50%
3. По типу объекта (исключение несовместимых категорий)
4. IQR-очистка выбросов (Q1 - 1.5*IQR ... Q3 + 1.5*IQR)
5. Медиана оставшихся цен за единицу площади

## 11. Cron расписание (VPS)

```
# Каждые 4 часа — парсинг
0 */4 * * *  cd /opt/torgi-proxy && source venv/bin/activate && python torgi_monitor.py --region perm
0 */4 * * *  cd /opt/torgi-proxy && source venv/bin/activate && python efrsb_monitor.py --region perm

# После парсинга — обогащение (цепочка)
# enrich_cadastral → enrich_market → enrich_scoring
```

Цепочка обогащения запускается после парсинга. Каждый скрипт обрабатывает только лоты без соответствующих данных (resume-поддержка).

## 12. Определение типа объекта (resolve_property_type)

Приоритет определения `property_type`:

1. **Название лота (name)** — наивысший приоритет:
   - "нежил" → commercial
   - "жилой дом", "коттедж", "дача" → house
   - "квартир", "комнат" (без "многоквартирн") → apartment
   - "помещен", "здани", "склад", "офис" → commercial
   - "земельн", "участ", "з/у" → land
   - "гараж", "бокс" → commercial

2. **Общий текст (name + category)** — если название не дало результат

3. **property_type из БД** — фаллбек

4. **По умолчанию** → commercial

Объекты с "права требования" или "дебиторская" пропускаются.
